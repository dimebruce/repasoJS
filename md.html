Variables

Es importante mencionar que no se necesita poner ; al final de la linea, porque JS es noble en ese sentido.

No se declara el tipo de dato de una variable como en otros lenguages, JS lo detecta por defecto.

Es de buenas practicas, usar camelCase para las variables

Tipos

  • var

  • let

  • const

var es poco usada, JS se inicio con ese tipo de variable, pero conforme pasó el tiempo se detecto vulnerabilidades y es donde let y const surgen.

let

Let tiene las siguientes ventajas

  • Puede reasignar un nuevo valor a la variable

  • Pueden declararse sin ningun valor

const

Const tiene las siguientes caracteristicas

  • No se pueden reasignar

  • No se pueden declarar sin ningun valor

Const es más utilizado que let, más que nada por la interacción con el DOM de HTML, pues este nunca cambia.

Rules de variables

Las variables no pueden:

  • Iniciar con números

  • … caracteres especiales

  • Guión bajo si está permitido

  • No pueden tener espacios

Object

Las ventaja de los objetos es que se puede ver como un arbol de viariables que tienen relacion entre ellas.

No sólo se puede acceder al valor de los objetos, ni insertarlos. También se puede sobreescribirlos de la siguiente manera.

Arreglos / Arrays

Los arreglos tiene algo de similitud a los objetos, sin embargo la sintaxis, junto con el acceso a sus valores es diferente.

Declarar un arreglos


const  tech = [100, 20, 'Angular', 'React']

console.table(tech)

Para acceder a una propiedad o valor en especifico se busca por el índice. 


// Índice 0 1 2 3

const  tech = [100, 20, 'Angular', 'React']

console.table(tech[3])

Queda destacar que el índice siempre empieza en cero, por eso, aunque nuestro valor sea el cuatro, contando a partir del cero, sería 3.

Si aplicamos un typeof al arreglo, no aparecerá que será de tipo array, sino object. 


console.table(typeof  tech[3])

Imprimir todo el arreglo como String


console.table(tech.toString())

Manipulación de Arreglos

Añadir elementos


const  tech = ['React', 'HTML', 'CSS', 'NodeJS', 'JS']

console.table(tech)

  

// Añadir elementos al array

console.table(tech)

  

tech.unshift('MongoDB') // Añade el valor al inicio del array

console.table(tech)

Forma optimizada para uso de React


const  nuevoArray = [...  tech, "MongoDB"] // Añade el valor al final del array

const  nuevoArray2 = ["MongoDB", ...  tech] // Añade el valor al inicio del array

Recordemos que los … crean una copia del array, así que el original se mantiene.

Eliminar elementos

Forma optimizada para uso de React


tech.splice(2,1)

console.table(tech)

Borrara el índice 2, y sólo la posición 1. Si no se pone el segundo número, borra todo lo que hay después del índice 2

Reemplazar valores dentro de un Arrays


tech[0] = 'PHP'

const  remplazarArray = tech.map( function (tech) {

if (tech  ===  'React') {

return  'PHP'

Destructuring con arreglos


const  tech = ['React', 'HTML', 'CSS', 'NodeJS', 'JS']

Es similar a la forma en como se hace con los objetos, sin embargo para extraer las posiciones, se usan los índices.

Como verás no se ve muy amigable, por eso son mejores los objetos en ese sentido.

Iteradores

El iterador es la forma de recorrer un arreglo el número de veces que sea su valor interno.

Es decir, si tenemos un array con 4 elementos o 3 índices, recordando que el cero se cuenta, el forEach, pasara 4 veces por el arreglo. 


const  tech = ['React', 'HTML', 'CSS', 'NodeJS', 'JS']

  

// forEach

  

tech.forEach(tech  => {

console.table(tech)

});

map

Map se comporta igual que el forEach


// Map

  

tech.map(tech  => {

console.table(tech)

});

La Diferencia de uno con otro recae en que map te permite llenar un nuevo arreglo con la iteración, mientras que forEach no, sólo itera. 


const  arrayMap = tech.map(tech  => {

return (tech)

});

console.table(arrayMap)

Funciones

El objetivo de una function declaration es ejecturar una tarea o realizarla.

Al igual que las variables, se deben respetar ciertas cosas como que el nombre de la function no debe empezar con

  • números

  • carácteres especiales

El ejemplo básico de cómo declarar una función es de la siguiente manera. 


// Funciones - Function Declaration

  

function  suma() {

console.log(5  +  5) //10

}

suma()

Se declara la función llamada suma, dónde no se le pasa ningun paramentro pero dentro del cuerpo, es decir, dentro de las llaves está la operación a realizar. En este caso es la suma de 5 + 5 y el resultado de la operación se manda a imprimir

Es importante conocer que la función se puede llamar N veces sea necesario. Así como no es necesario que se mande a llamar después de crearla. 


suma()

  

function  suma() {

console.log(5  +  5) //10

}

En este caso JS no es como otros lenguajes, aquí no marca err y la función seguirá funcionando normal,. A esto se le llama Hoisting

Parametros dentro de Funciones 


function  suma(5, 15) {

console.log(num1  +  num2)

}

suma(5, 15)

Lo que está dentro del parentesis suma(5, 15), se conoce como párametros.

Lo que está entre las llaves es el cuerpo de la función.

Y por último cuando se llama a la función suma(5, 15) se le conoce como argumentos.

¿Qué pasa si en vez de pasar dos argumentos se pasa 1, pero la función está definida por dos parámentros. 


function  suma(num1, num2) {

console.log(num1  +  num2)

}

suma(150) //NaN

El resultado será NaN Not a number, porque la función espera dos números.

Ahora, la función se le puede asignar un valor por defecto, y así pasarle sólo un argumento aunque espere dos parámetros la función. 


function  suma(num1, num2 = 10) {

console.log(num1  +  num2)

}

suma(150) // 160

La función recibe el argumento, y cómo detecta que sólo hay una párametro llenado, toma el segundo por default y la operación se realiza.

Se puede poner un valor por default y no usarlo.


function  suma(num1, num2 = 10) {

console.log(num1  +  num2)

}

suma(150, 150) // 300

La salida de esta operación es 300, pueto como si hay dos argumentos, omite el valor por default dentro del paramentro y sigue con la operación.

Return dentro de las funciones

Funciones que retornan valores…


function  suma(num1, num2 = 10) {

return  num1  +  num2

}

const  resultado = suma(150, 150)

console.log(resultado) // 300

En esa operación recibe los argumentos, los pasa como paramentros, y retorna la operación.

Se crea una variable llamado resultado la cual aloja la operación y por último lo mandamos a imprimir.

Function Expression

Este es el segundo modo de las funciones, la cuál tiene dos grandes diferencias notorias.

  • Su sintaxis

  • No se puede mandar a llamar antes de que se creó la función.

sintaxis


// Function declaration

function  suma(num1, num2 = 10) {

console.log(num1  +  num2)

}

suma(150, 150) // 300

  

// Function Expression

const  sumar = function (num1, num2) {

console.log(num1  +  num2)

}

  

sumar(150, 150) // 300

Como vemos la sintaxis es diferente, la segunda guarda la función dentro de una variable. Fuera de eso podemos hacer casi lo mismo, podemos poner valores por default dentro de los argumentos, y podemos hacer el retorno dentro del cuerpo.

El llamado

A diferencia de la function declaration, la function expression sólo puede ser llamada después de haberla creado de lo contrario mandaría error. 


// ❌

sumar(150, 150) // 300

  

const  sumar = function (num1, num2) {

console.log(num1  +  num2)

}

  

// ✅

const  sumar = function (num1, num2) {

console.log(num1  +  num2)

}

  

sumar(150, 150) // 300

Arrow Functions

Difenrencias entre una function expression y un arrow function

Sintaxis:


// F. Expression

const  sumar = function (num1, num2) {

console.log(num1  +  num2)

}

  

sumar(150, 150) // 300

  

// Arrow F.

const  sumar = (num1, num2) => {

console.log(num1  +  num2)

}

  

sumar(150, 150) // 300

Se elimina la palabra function y se agrega un => arrow (flecha) después del parentesis.

Todas estás funciones heredan los beneficios de las F. Expression, sin embargo hay algo más. Pues cuando sólo tienes una linea de código dentro del cuerpo de la función puedes expresarla de la siguiente manera. 


// Arrow F.

const  sumar = (num1, num2) => {

return  num1  +  num2

}

  

sumar(150, 150) // 300

  

// Arrow F. Simplificada

const  sumar = (num1, num2) =>  num1  +  num2

  

sumar(150, 150) // 300

Se nota que se puede elimar las llaves, y también el return, porque se entiende con esta nueva sintaxis que el return va implicito.

Sólo funciona con una sóla linea de código, de lo contrario debes agregar las llaves y el return funciona igual (implicito)

También cuando tienes dos parámetros dentro de la función si debes poner los parentesis 


const  sumar = (num1, num2) =>  num1  +  num2

Pero cuando sólo hay 1 parametro te los puedes ahorrar


const  sumar = num  =>  num  +  15

Los beneficios de los Arrow Functions se ve más claramente en el uso de Arrays.

Arrow F. con Arreglos.


const  tech = [100, 20, 'Angular', 'React']

  

// Con Arrow Function

  

const  remplazarArray = tech.map( tech  => {

if (tech  ===  'React') {

return  'PHP'

} else {

return  tech

}

} )

  

console.table(remplazarArray)

Recordemos que map nos ayuda a iterar por cada una de las posiciones del arreglo, entonces con ayuda del arro función se crea la función donde si se encuentra un valor llamado React, lo va a cambiar por PHP

Otro ejemplo pero con filter


const  filtrarArray = tech.filter( tech  =>  tech  ===  'React' )

console.table(filtrarArray)

Aplicamos nuestro arrow function, dónde por medio de filter vamos a buscar la palabra React, y si la encontramos la regresamos como arreglo.

Nota: Es importante recordar que el método filter retorna un arreglo 


const  filtrarArray = tech.filter( tech  =>  tech  !==  'React' )

console.table(filtrarArray)

Ahora hacemos lo mismo, arrow function pero esta vez vamos a devolver todo lo que sea diferente a la palabra React, recordemos el operador lógico diferente a (!==).

Array Methods más útiles

Includes

Includes comprueba si un elemento existe en el array


const  tech = [100, 20, 'Angular', 'React']

const  resultado = tech.includes('Laravel')

console.table(resultado) // false

Some

Some devuelve su al menos uno cumple la condición pero devuelve un valor booleano true/false y NO nos dice que elemento es. 


const  numbers = [10, 30, 50]

const  resultado = numbers.some( numbers  =>  numbers  >  15)

console.table(resultado) // true

Find

Find devuelve el primer elemento que cumpla con la condición.


const  numbers = [10, 30, 50]

const  resultado = numbers.find( numbers  =>  numbers  >  15)

console.table(resultado) // 30

Every

Retorna true o false si todos cumplen con la condición


const  numbers = [10, 30, 50]

const  resultado = numbers.every( numbers  =>  numbers  >  15)

console.table(resultado) // false

Reduce

Suma todos los valores dentro del array, ideal para un carrito de compras. el Reduce remplaza al forEach 


const  numbers = [10, 30, 50]

const  resultado = numbers.reduce( (total, numero) =>  total  +  numero, 0) //

console.table(resultado) // 90

Después del arrow, o sea en el cuerpo de la función, debemos establecer que numero (los números que se van iterando uno a uno) se le debe sumar a total, y el 0 que definimos es lo que indicamos como valor inicial.

En el ejemplo el resultado sería 90, pero si hubieramos puesto 100 como valor inicial, el resultado hubiera sido 190

Condicionales

Algunos operadores lóicos a tener en cuenta son:

| Descripción | Simbolo |

| :----: | :----: |

| Menor que | < |

| Mayor que | > |

| Menor igual que | <= |

| Menor igual que | >= |

| Diferente que | != |

| Igual a | == |

| Estrictamente igual a | === |

| Or / o | || |

| And / y | && |

if

Las condicionales son fragmentos de código que básicamente su función es ver si una condición se cumple ejecuta un código, sino otra. 


let  fondo = 5000

let  retiro = 100

  

if (fondo  >  retiro) {

// Se puede retirar dinero

console.log("Excelente, si puedes retirar")

} else {

// No se puede retirar

console.log("Lo siento, no puedes retirar dinero")

}

Se plantea un ejemplo de un ATM o cajero de banco donde el usuario tiene un fondo que simula la cantidad de dinero disonible en su cuenta, y segundo tiene un monto a retirar.

Ambas variables se meten dentro de una condicional if donde se evalua que si el fondo es mayor a lo que el usuario piensa retirar le permite continuar con la operación, de lo contrario le rechazará el retiro.

Otro ejemplo que nos ayudará a entender mejor las condicionales.


console.log("**Banco de Bruce**")

  

const  saldo = 600

const  pagar = 400

const  tarjeta = true

  

if (saldo  >  pagar) {

console.log("Puedes pagar");

} else  if (tarjeta  ==  true) {

console.log("Puedes pagar")

} else {

console.log("No puedes pagar.")

}

Reduciendo código


console.log("**Banco de Bruce**")

  

const  saldo = 600

const  pagar = 400

const  tarjeta = true

  

if (saldo  >  pagar) {

console.log("Puedes pagar");

} else  if (tarjeta) {

console.log("Puedes pagar")

} else {

console.log("No puedes pagar.")

}

En el else if se asume que la condición se cumple, por eso no es necesario poner la asignación tarjeta == true

Reduciendo más el código


console.log("**Banco de Bruce**")

  

const  saldo = 600

const  pagar = 400

const  tarjeta = true

  

if (saldo  >  pagar  ||  tarjeta) {

console.log("Puedes pagar");

} else {

console.log("No puedes pagar.")

}

If ternario

Es básicamente un if normal, pero en una sóla linea. Acá la diferencia. 


const  autenticado = true

if (autenticado) {

console.log("Si está autenticado");

} else {

console.log("No está autenticado");

}

Ahora con if ternario 


const  autenticado = true

autenticado  ?  console.log("Si esta autenticado") :  console.log("No está autenticado")

Nota: También puede haber if ternarios dentro de otros if ternarios.

Scope de variables

El alcance de las variables tiene que ver sobre dónde puedes ocupar tus variables. Recordemos que en JS no puedes declarar dos variables con el mismo nombre, pero cuando ocupas variables cambia un poco la situación. 


const  precio = 600

  

function  scope() {

const  precio = 1500

console.log(precio);

}

scope() // 1500

Como se ve hay dos variables nombradas precio, pero cuál será el resultado a imprimir? Fácil: 1500 puesto que la variable precio dentro de la función tiene prioridad sobre la global que está fuera. 


const  precio = 600

  

function  scope() {

console.log(precio);

}

scope() // 600

En este caso, como no existe una variable local, toma la que viene de afuera (global), y por eso el resultado es 600

querySelector

Accediendo a una clase del DOM / document desde JS


const  heading = document.querySelector(".heading")

console.log(heading);

Se genera una variable con el mismo nombre de la clase que se quiere obtener, se le asigna por un = al document dónde buscaremos nuestra clase y por medio del querySelector accedemos a la clase ‘.heading’.

Accediendo a un id del DOM / document desde JS


const  heading = document.querySelector("#heading")

console.log(heading);

Se hace de la misma manera, solo que despupes del querySelector se pone un # en vez de un ..

Nota: Es importante conocer que querySelector sólo te va a retornar un elemento.

querySelectorAll

A diferencia de querySelector, en este traemos toda la colección que contenga nuestra selelección.

Si en el documento (html) tenemos


<nav  class="nav nav-masthead justify-content-center float-md-end">

<a  class="nav-link"  aria-current="page"  href="index.html">Inicio</a>

<a  class="nav-link active"  href="DOM.html">DOM</a>

</nav>

y en el JS


const  navegacion = document.querySelectorAll("#navegacion a")

console.log(navegacion);

Como vemos, aplicamos un querySelectorAll a navegación con el id navegacion y con el enlace (Anchor ) como hijo. Eso nos arrojará un arreglo al momento de mandarlo a imprimir con el console.log.

Modificando el DOM desde JS


const  heading = document.querySelector("#heading")

heading.textContent = "Un nuevo titulo desde JS 😲"

Para cambiar un valor de nuestro HTML pero desde JS, lo debemos llamar con un querySelector y despues le asignamos con el nombre de la variable lo concatenamos y le asignamos el nuevo texto.

Eventos del DOM

Los eventos son metodos que nos otorga JS que están en espera de una acción.

Evento de click


const  heading = document.querySelector("#heading")

  

heading.addEventListener('click', () => {

heading.textContent = "Nuevo titulo a un click 🧙‍♂️"

})

Llamaos el id llamado heading, lo guardamos en una variable con el mismo nombre.

Después concatenamos la variable recien creada con el evento addEventListener, dentro de ese método indicamos que usaremos el evento ‘click’ y después ejecutamos un arrow function anónima (sin parametros) el cuál tendrá en el cuerpo de la función asignarle el nuevo titulo Nuevo titulo a un click 🧙‍♂️ cuando el usuario le de click en el DOM.

Evento de inputs

Cómo detectar lo que está escribiendo el user dentro de un input


const  name = document.querySelector("#name")

  

name.addEventListener('input', (e) => {

console.log(e.target.value);

})

Llamamos nuestro id del DOM a JS, lo asignamos a nuestra variale name, después creamos nuestro evento, le decimos que será del tipo submit y creamos nuestra arrow function, donde le pasaremos el parametro e de event y mandamos a imprimir a consola todo lo que el user está escribiendo en el DOM.

Ahora se puede jugar con esto de muchas maneras, sin embargo hagamos otro ejemplo 


const  password = document.querySelector("#password")

password.addEventListener('input', mostrarPassword)

  

function  mostrarPassword(){

password.type = "text"

  

setTimeout(() => {

password.type = "Password"

}, 1000);

}

Traemos el id, lo asignamos y creamos nuestro addEventListener concatenado a la variable que hicimos la cual indicamos que será de tipo submit y ejecutaremos una función cuando este pase.

mostrarPassword será la función a ejecutar, que otras palabras lo que hace es que el input del DOM con type password, lo convierte en text cuando se ejecuta, pero despúes de 1 segundo con el setTimeout lo convierte a type password nuevamente.

Ahora, podemos agregar validaciones sencillas para que el user pueda si o si llenar el form para que se mande la data. 


const  form = document.querySelector('.formulario')

form.addEventListener('submit', e  => {

e.preventDefault() // Previene la acción por default

  

// Chaining .value porque trae el valor que contiene ese input

const  name = document.querySelector("#name").value

const  password = document.querySelector("#password").value

  

if (name  ===  ''  ||  password  ===  '') {

console.log("Todos los campos son obligatorios");

} else {

}

})

Vemos un if donde pregunta que si el input name o password es igual a vacio, entonces mandará una alerta diciendo que todos los campos son obligatorios, de lo contrario se manda la data por el submit y se imprime el mensaje Enviando la data …

Agregando HTML desde JS


const  form = document.querySelector('#formulario')

form.addEventListener('submit', e  => {

e.preventDefault() // Previene la acción por default

  

// Chaining .value porque trae el valor que contiene ese input

const  name = document.querySelector("#name").value

const  password = document.querySelector("#password").value

  

const  alertPrevia = document.querySelector('.alert')

if (alertPrevia) {

alertPrevia.remove()

}

  

const  alert = document.createElement('DIV')

alert.classList.add('alert')

  

if (name  ===  ''  ||  password  ===  '') {

alert.textContent = "Todos los campos son obligatorios"

alert.classList.add('alert', 'alert-danger')

} else {

alert.textContent = "Enviando la DATA ..."

alert.classList.add('alert', 'alert-success')

}

  

form.appendChild(alert)

console.log(alert);

})

El código parece grande, pero es más simple de lo que se ve.

Lo único nuevo aquí al code anterior es que generamos un div


const  alert = document.createElement('DIV')

alert.classList.add('alert')

Se genera el div en mayúsculas porque es de buenas practicas, al final el DOM lo interpreta y lo hace minuscula, en fin, se genera con createElement concatenado al documento.

Asignamos una clase con la variable creada, concatenada con classList y add y le pasamos alert. Ahora en el el if mandamos el texto a imprimir segun sea el caso con el textContent y lo pintamos con alert-danger o alert-success.

Ahora, para mandar el div que creamos al DOM ocupamos la siguiente linea. 


form.appendChild(alert)

Y listo, así se manda una etiqueta HTML, con texto y estilos desde JS

Imports y Exports

La intención de esta sección es comprender que se tiene que optimizar el código lo más que se pueda, para este caso ocuparemos una función creada en un archivo JS nombrado suma.js el cual tendrá dicha función sólo en ese archivo pero lo usaremos en el archivo 28.js para mandarlos al index.html

Exportando

archivo suma.js


function  suma(num1 = 0, num2 = 0) {

return  num1  +  num2

}

export  default  suma

Vemos una función simple, que tiene 2 paramentros y que retorna la suma de los dos números.

Por último, exportamos la función suma con el default.

El default lo ocupamos cuando sólo tenemos una función a exportar

Importando

Vamos a nuestros archivo 28.js


import  comoquieras  from  "./suma.js";

const  resultado = comoquieras(20, 30)

console.log(resultado); // 50

Importamos, con la palabra reservada import le ponemos la palabra o variable que nosotros queramos porque en el export está como default y le decimos la ruta de nuestro archivo.

Creamos la variable resultado donde guardaremos el resultado de la función comoquieras(20,30)

Y lo mandamos a imprimir por el log a resultado.

Nota: Esto nos va a generar un error, llamado: “Uncaught SyntaxError: Cannot use import statement outside a module”

El cual se soluciona yendo al index.html, donde importamos el script 28.js

Pasar de esto


<script  src="./js/28.js"></script>

A esto:


<script  src="./js/28.js"  type="module"></script>

Nota: Agrega el type module dentro del script del HTML

Y con ello tendremos el resultado de la suma en la consola.

Exportando e importando con más de una función

Todo lo que vimos anteriormente fue con una sola función, pero qué pasa si nuestro archivo tiene más de una función …

archivo suma.js


function  suma(num1 = 0, num2 = 0) {

return  num1  +  num2

}

function  resta(num1 = 0, num2 = 0) {

return  num1  -  num2

}

export {

suma,

resta

}

archivo 28.js


import {suma, resta} from  "./suma.js";

  

const  resultadoSuma = suma(20, 30)

const  resultadoResta = resta(20, 30)

  

console.log(resultadoSuma);

console.log(resultadoResta);

Analicemos, en suma.js lo que hicimos es un nuevo export pero en forma de objeto {}

donde ponemos suma, resta en el cuerpo.

Recordemos que se puede omitir el segundo valor del objeto, siempre y cuando el nombre sea igual, así que por eso pasamos de esto: 


...

  

export {

suma: suma,

resta: resta

}

A esto:


...

  

export {

suma,

resta

}

En el archivo 28.js. importaremos igual, pero haciendo el Destructuring Object ya visto en este repaso. 


import {suma, resta} from  "./suma.js";

En el cual importamos suma y resta, es importante que aquí si ocupemos el nombre que tenemos donde se declaró la función, de lo contrario no funcionará. Y el resto se lee por sí solo.

Fecth es como el nuevo AJAX. Es la parte que te ayuda a consumir una API de otro servidor para tu página. 


const  url = "https://rickandmortyapi.com/api/character"

.then( (respuesta) => {

cunsultarAPI()

Nota: Los then es la forma visual de que existe una promesa.

Creamos una variable llamada url, donde alojaremos el link de la API.

Después creamos una función dónde tendremos un fetch de la url creada.

Ahora, empezamos con la promesa .then donde si existe, cree toda la data que viene con ella con un formateo .json

Y por último mandamos a imprimir el resultado.

Nota: Es importante saber que las promesas tienen dos estados; respuesta y resultado.

Haciendo más chico el código.


const  url = "https://rickandmortyapi.com/api/character"

.then( respuesta  =>respuesta.json() )

cunsultarAPI()

Cambiando de Promises a Async y Await 


const  url = "https://rickandmortyapi.com/api/character"

  

const  cunsultarAPI = async () => {

const  respuesta = await  fetch(url)

const  resultado = await  respuesta.json()

console.log(resultado);

}

cunsultarAPI()

A simple vista, async y await nos genera menos código y con el mismo resultado.

La diferencia más poderosa es saber que nuestra función se vueve asincrona, lo que significa que por cada await nuestro código no se ejecutará hasta que no termine con el await en marcha.

Conexiones mmúltiples con async y await 


const  url = "https://rickandmortyapi.com/api/character"

const  url2 = "https://rickandmortyapi.com/api/episode"

  

const  cunsultarAPI = async () => {

const  respuesta = await  fetch(url)

const  resultado = await  respuesta.json()

console.log(resultado);

  

const  respuesta2 = await  fetch(url2)

const  resultado2 = await  respuesta2.json()

console.log(resultado2);

}

  

cunsultarAPI()

Vemos que existen dos url con rutas difentes, el consumo es el mismo. Sin embargo hay problema no notorio para este tipo de links genericos, pero si posibles para producción.

Y es que al hacerlo de esta manera, el consumo de la primera url, toma su tiempo y lo hace por partes como ya habíamos dicho, y eso hace que el consumo de la segunda url quede en espera y eso está mal. Porque el segundo consumo no depende del primero.

Iniciando el consumo en paralelo 


const  cunsultarAPI2 = async () => {

const [respuesta, respuesta2] = await  Promise.all([fetch(url), fetch(url2)])

const  resultado = await  respuesta.json()

const  resultado2 = await  respuesta2.json()

console.log(resultado);

console.log(resultado2);

}

Está es la forma eficiente de consumir dos APIs que no dependen de sí mismas, ya que las dos se ejecutan al mismo tiempo y la respuesta se guarda dentro de un array destructuring.

Nota: Puede que una respuesta tarde más que otra, eso dependerá del tamaño de la API.

Por último traemos el resultado y lo formateamos con respuesta.json()

Performance

Existe un método de JS llamado Performance el cual permite el cronometraje de alguna ejecución. 


cunsultarAPI()

  

const  cunsultarAPI2 = async () => {

const  inicio = performance.now()

  

const [respuesta, respuesta2] = await  Promise.all([fetch(url), fetch(url2)])

const  resultado = await  respuesta.json()

const  resultado2 = await  respuesta2.json()

console.log(resultado);

console.log(resultado2);

const  fin = performance.now()

console.log(`Tiempo de Segundo ${fin-inicio}`);

}

  

cunsultarAPI2()

Sólo basta con ir a la console y comprobar qué método es más rapido.

Template Strings

El uso de esto es básicamente concatenar variables con strings o funciones. 


const  producto = "Computadora Alienware"

const  precio = 100

const  color = "Roja"

  

console.log("Producto: "  +  producto  +  ", Precio: "+  precio  +  ", Color: "+  color  +  ".");

Está era una de las formas de imprimir el valor de las variables otorgandole un pequeño formateo entre ellas, también existe la siguiente: 


console.log("Producto: " , producto , ", Precio: ", precio , ", Color: ", color , ".");

De la misma forma, pero en vez del símbolo de plus (+) se usa la coma (,)

Pero ninguna de las dos formas es funcional. Para ello se creó el Template String 


console.log(`Producto: ${producto}, Precio: ${precio}, Color: ${color}.`);

Es más eficiente y más clara. ☺
